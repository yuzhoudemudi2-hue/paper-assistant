# Codex Development Instructions

> This document defines the mandatory development rules for the Paper Assistant project.
>
> Every source file, test, script, configuration, and documentation generated by Codex MUST comply with this document.
>
> Unless the user explicitly requests otherwise, these instructions always take precedence over convenience.

---

# 1. Project Mission

Paper Assistant is a long-term intelligent research assistant designed for researchers working in General Relativity and High Energy Theory.

Its primary objective is to automatically discover newly submitted papers from arXiv, evaluate their academic importance through citation analysis and AI models, summarize the scientific contributions, translate important content into Simplified Chinese, and deliver professional reports to users every day.

The project is expected to operate continuously for many years with minimal manual maintenance.

Therefore every design decision must prioritize:

- correctness
- maintainability
- extensibility
- readability
- robustness
- reproducibility

instead of short-term implementation speed.

Every generated source file should be considered production-quality software.

Codex should always assume that the project will continue growing over many years.

Never optimize for temporary solutions if a cleaner architecture exists.

When multiple implementations are possible, choose the solution with better long-term maintainability.

The project follows professional software engineering practices instead of prototype development.
